## 📋 问题诊断

### 原始问题
1. `@tailwindcss/postcss` 缺失
2. `@solana/wallet-adapter-react-ui/styles.css` HMR 错误
3. Next.js 应用无法启动

### 根本原因
- PostCSS 配置使用了 Tailwind v4 的 `@tailwindcss/postcss`，但项目需要 Tailwind v3
- 缺少 Tailwind CSS 配置文件
- globals.css 没有正确导入 Tailwind 指令
- next.config.js 使用了已废弃的 `images.domains` 配置

---

## 🔧 修复步骤

### 1. 安装正确的依赖
```bash
npm install tailwindcss postcss autoprefixer --save-dev
```

### 2. 更新 PostCSS 配置
**文件**: `postcss.config.mjs`
```javascript
const config = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
};
export default config;
```

### 3. 创建 Tailwind 配置
**文件**: `tailwind.config.js`
```javascript
module.exports = {
  content: [
    './pages/**/*.{js,ts,jsx,tsx,mdx}',
    './components/**/*.{js,ts,jsx,tsx,mdx}',
    './app/**/*.{js,ts,jsx,tsx,mdx}',
  ],
  theme: { extend: {} },
  plugins: [],
}
```

### 4. 更新全局样式
**文件**: `app/globals.css`
- 添加 Tailwind 指令：`@tailwind base/components/utilities`
- 保留自定义样式
- 钱包适配器样式已在 `providers.tsx` 中导入

### 5. 更新 Next.js 配置
**文件**: `next.config.js`
- 将 `images.domains` 更新为 `images.remotePatterns`（Next.js 16 推荐）

### 6. 清理并重启
```bash
rm -rf .next
npm run dev
```
